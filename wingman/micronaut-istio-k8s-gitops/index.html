Kubernetes 上的基于 Micronaut 的云原生项目

介绍

本项目使用的是一台 Intel NUC 8 主机：

  • CPU：i5-8260U 4C8T
  • Memory：64GB
  • OS：Ubuntu 22.04

本项目所有应用、服务均部署于 Kubernetes 集群中，使用 Helm 或 Operator 或 Kustomize 的形式安装、部署。

Micronaut

Micronaut 为 JVM 平台上新一代的 IoC 框架，同时支持 Java、Groovy、Kotlin 编程语言，使用编译时进行依赖注入的实现，实现运行时的快速启动、低内存消耗。

本项目使用 Micronaut 作为微服务基础框架，Groovy 作为开发语言。将 Project Reactor 用于 HTTP Server 和数据库访问操作，实现全面的响应式编程。

以 Micronaut 为中心，接入了以下服务、工具：

  • Keycloak：认证与鉴权服务，实现统一登录
  • Jaeger：分布式跟踪
  • Prometheus：用于应用运行监测
  • Flyway：数据库模式与数据版本跟踪
  • R2dbc：响应式关系数据库访问操作
  • Elasticsearch：全文搜索数据库与应用日志存储

GitOps

以 GitLab 为中心，通过 Merge Request 事件通知 Tekton。

Tekton 作为云原生的流水线工具，在 Kubernetes 中启动 Pod 来执行应用测试、打包、部署等任务，具有良好的可扩展性，健壮性等。

通过 ArgoCD 与 Kustomize，将应用的声明式配置自动持续部署到 Kubernetes 集群中，避免了人工手动部署的不确定性，通过声明式配置，降低了持续部署的复杂性。

Istio

通过 Istio 服务网格，为应用提供云原生的微服务能力。

为应用提供流量管理，安全管控，可观测性。

Observability

使用 Prometheus 采集应用指标。

使用 Jaeger 分布式跟踪微服务。

使用 Fluent-bit 采集应用日志并转发至 Elasticsearch

Flink CDC

使用 Flink CDC 读取 MySQL binlog，将数据实时处理并存储至 Elasticsearch。

将应用中需要多表关联全文检索的操作转移到 Elasticsearch 中。
