User Guide 小结

初始化 Gradle 项目

使用 init task, 查看 init plugin 获取更多信息

为项目生成 wrapper 脚本并固化 Gradle 版本

内置的 gradle wrapper task 生成一个脚本 gradlew, 这个脚本调用了一个指定版本的 Gradle, 会在使用这个指定版本的 Gradle 之前下载它

除了 --gradle-version, 还可以指定 --distribution-type=(bin|all), --gradle-distribution-url, --gradle-distribution-sha256-sum

--distribution-type=all 可以让 IDE 开启代码补全并且可以导航到 Gradle 源代码

命令行选项 dry run

命令行选项 --dry-run

在停止执行所有 task 的 action 的情况下运行 Gradle

主要用来显示哪个 task 会执行, 查看 task 的执行顺序

持续构建 Continuous Build

在 task 的 inputs 改变的时候重新运行这个 task

在各个平台上面都有各种问题, 建议不使用持续构建

终端命令补全

Gradle 提供了 bash 和 zsh 的命令行补全

gradle-completion

非 HTTPS 协议的仓库地址

使用 --insecure-protocol 选项

  • fail : 如果有不安全的仓库 URL 就马上中断构建
  • allow : 自动为生成的 Gradle 构建脚本中的 Maven 仓库 URL 设置 allowInsecureProtocol 属性为 true
  • warn
  • upgrade : 将 URLs 从 http 转换为 https

SourceSet

A SourceSet represents a logical group of Java source and resources files

一个 SourceSet 代表着一个 Java 源码和资源文件的逻辑集合

Java Plugin 的 annotationProcessor

Since implementation details matter for annotation processors, they must be declared separately on the annotation process path

由于实现细节对注释处理程序很重要，所以必须在注释处理路径上分别声明它们

Gradle ignores annotation processes on the compile classpath

Gradle 忽略了编译类路径上的注释过程
