Running Applications with the CLI

使用 run 命令可以直接编译并运行 Groovy 源代码

Example - hello.groovy:

使用如下命令编译并运行应用: $ spring run hello.groovy

要传递命令行参数给应用, 先使用 -- 分隔 spring 命令的参数: $ spring run hello.groovy -- --server.port=9000

使用 JAVA_OPTS 环境变量来设置 JVM 命令行参数: $ JAVA_OPTS=-Xmx1024m spring run hello.groovy

Custom Dependency Management

默认情况下, 解析 @Grab 依赖时, CLI 使用定义在 spring-boot-dependencies 中的依赖管理 (dependency management)

额外的 dependency management, 用来覆盖默认的 dependency management, 可以使用 @DependencyManagementBom 注解进行配置

在任何可以使用 @Grab 的地方都可以使用 @DependencyManagementBom

在应用中至多使用一次 @DependencyManagementBom

Initialize a New Project

init 命令使用 start.spring.io 创建一个新项目

$ spring init <命令选项> 项目目录

将会在 项目目录 中创建项目

Option Description
-a --artifact-id <String> 指定项目坐标 (coordinates); 推断 archive name (例如 wallet-server)
-b --boot-version <String> 指定 Spring Boot 版本 (例如 1.2.0.RELEASE)
--build <String> 指定构建系统 (maven 或 gradle) (默认 maven)
-d --dependencies <String> 指定项目使用的依赖, 使用逗号进行分隔 (例如 -d=web,data-jpa)
--description <String> 指定项目描述
-f --force 强制覆盖已存在的文件
--format <String> 生成的内容的格式 (project 或 build) (build 即只生成构建文件 pom.xml 或 build.gradle) (默认 project)
-g --group-id <String> 设置项目的 Group Id (例如 org.daisy)
-j --java-version <String> 设置 Java 版本 (例如 1.8)
-l --language <String> 指定编程语言 (默认 java) (例如 -l=groovy)
--list 列出服务的功能
-n --name <String> 指定项目名称; 推断语言名称
-p --packaging <String> 项目打包类型 (例如 jar)
--package-name <String> 指定包名 (例如 org.daisy)
-t --type <String> 项目类型. 不常使用
--target <String> 指定 service 的 URL (默认 https://start.spring.io)
-v --version <String> 指定项目版本 (例如 0.0.1-SNAPSHOT)
-x --extract 解压缩项目. 如果坐标没有指定扩展名则进行推断

Adding Extensions to the CLI

通过使用 install 命令可以添加扩展到 CLI 中

install 接收一个或多个格式为 group:artifact:version 的构件坐标

例如: $ spring install com.example:spring-boot-cli-extension:1.0.0.RELEASE

除了通过坐标安装你提供的构件, 这个构件所有的依赖也会被安装

使用 uninstall 命令来卸载依赖: $ spring uninstall com.example:spring-boot-cli-extension:1.0.0.RELEASE

将会卸载这个构件和它的依赖项

使用 --all 选项来卸载所有的依赖: spring uninstall --all

Configuring the CLI with settings.xml

Spring Boot CLI 使用 Maven 解析器, Maven 的依赖解析引擎, 来解析依赖

CLI 使用 ~/.m2/settings.xml Maven 配置来配置 Maven 解析器

CLI 支持一下配置:

  • Offline
  • Mirrors
  • Servers
  • Proxies
  • Profiles
    • Activation
    • Repositories
  • Active profiles
