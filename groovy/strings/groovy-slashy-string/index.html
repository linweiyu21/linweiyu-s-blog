Groovy Slashy string

Slashy strings 可以被认为是另一种定义 GString 的方式，不过使用了不同转义规则

Slashy strings 因此也支持字符串插值: /a ${color} car/

在 Slashy string 中不需要转义 /

特殊的地方

空的 Slashy string // 会被解析器认为是注释符

像 $() 或 $5 这些在 GString 中是错误的字符是可以正常工作在 Slashy string 中的，因为 Slashy string 是被设计用来更容易地创建正则表达式

在 Slashy string 中转义其实是不支持的，例如 println /\t/ 的输出结果是 \t ，转义只有在遇到 / 时才会生效，例如 /\/folder/ 的输出结果是 /folder

Slashy string 中不能使用 \ 作为一个字符串的结尾，因为这个最后的字符 \ 将会转义最后一个 /；如果必须要使用 \ 作为一个字符串的结尾，那么可以这样做: /ends with slash ${'\'}/；不过最好还是避免这样用 Slashy string
