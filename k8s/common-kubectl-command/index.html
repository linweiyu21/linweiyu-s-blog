常用的 kubectl 命令

命名空间

每个命名空间包含了一组对象

默认情况下, kubectl 交互的命名空间是 default

使用 --namespace 标志指定命名空间: kubectl --namespace=mystuff 操作 mystuff 命名空间中的对象

操作所有命名空间, 指定 --all-namespaces 标志

查看 Kubernetes API 对象

获取当前命名空间中的所有资源的清单: $ kubectl get <资源名称>

获取特定资源: $ kubectl get <资源名称> <对象名称>

获取更多信息, 使用 -o wide 标志

输出 JSON 或 YAML 格式的对象, 使用 -o json,-o yaml 标志

删除输出中的表头, 使用 --no-headers 标志

使用 JSONPath 查询返回对象特定字段: $ kubectl get pods my-pod -o jsonpath --template={.status.podIP}

查看特定对象的详细信息: $ kubectl describe <资源名称> <对象名称>

创建, 更新和销毁 Kubernetes 对象

创建对象: $ kubectl apply -f obj.yaml

再次运行 apply 命令更新对象: $ kubectl apply -f obj.yaml

apply 只会在集群对象与当前对象有差异时才进行操作与更新, 如无差异时则不会更改任何对象

如果你想确保集群状态与文件系统状态相匹配, 可以反复使用 apply 来协调状态

只查看 apply 命令会执行哪些操作, 使用 --dry-run 标志将对象输出到终端

使用 edit 命令交互式在线修改保存对象: $ kubectl edit <资源名称> <对象名称>

使用 edit-last-applied,set-last-applied,view-last-applied 来处理 apply 命令以前的配置历史记录: $ kubectl apply -f myobj.yaml view-last-applied

删除对象: $ kubectl delete -f obj.yaml

使用资源类型和名称删除对象: $ kubectl delete <资源名称> <对象名称>

给对象添加标签和注释

使用 annotate 和 label 命令来更新 Kubernetes 对象上的标签和注释: $ kubectl label pods bar color=red

使用 --overwrite 标志来覆盖已有的标签或注释

使用 <标签名称>- 的语法来移除标签: $ kubectl label pods bar color-

调试命令

查看正在运行的容器的日志: $ kubectl logs <Pod 名称>

使用 -c 标志选择要查看的容器

使用 -f 查看日志持续输出

使用 exec 命令, 在运行的容器中执行命令: $ kubectl exec -it <Pod 名称> -- bash

使用 attach 命令, 将终端附着到正在运行的进程上: $ kubectl attach -it <Pod 名称>. 这个命令允许你将输入发送到正在运行的进程, 前提是该进程能够读取标准输入

使用 cp 命令将文件复制到容器, 或从容器复制文件: $ kubectl cp <Pod 名称>:</path/to/remote/file> </path/to/local/file>

使用 port-forward 命令将网络流量从本地计算机转发到 Pod: $ kubectl port-forward <Pod 名称> 8080:80

使用 top 命令查看节点或 Pod 正在使用的资源列表: $ kubectl top nodes, $ kubectl top pods

使用 --all-namespaces 标志查看集群中所有节点或 Pod 的资源使用情况

自动补齐命令

要先安装 bash-completion 软件包才能启用命令自动补齐功能

在终端中临时启动该功能: source <(kubectl completion bash)

要自动启用该功能, 将它添加到 ${HOME}/.bashrc 文件中: echo "source <(kubectl completion bash)" >> ${HOME}/.bashrc
